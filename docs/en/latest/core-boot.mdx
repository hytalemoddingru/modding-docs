## Summary
Boot sequence, lifecycle events, and shutdown flow for HytaleServer.

## Boot sequence
- Main loads early plugins and may enable a transforming classloader
- LateMain configures logging, parses CLI options, constructs HytaleServer
- HytaleServer.boot sets up commands, plugins, assets, and universe

## Lifecycle events
- LoadAssetEvent validates assets
- BootEvent is dispatched once the universe is ready
- ShutdownEvent is dispatched on shutdown

## Shutdown
- Shutdown reasons tracked via ShutdownReason
- Plugins are stopped before network listeners close
- Config is periodically saved on a schedule
